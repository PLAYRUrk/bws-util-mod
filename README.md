# BWS Util Mod

Клиентский Forge-мод (`modId`: `bws_util`) для `Minecraft 1.21.11`, который добавляет:
- баллистический прицел для лука с пристрелкой (zeroing),
- режим автосопровождения цели (AUTO),
- тактический BedWars-оверлей (враги, команда, оценка силы, статус кроватей).

## Совместимость

- Minecraft: `1.21.11`
- Forge: `61.1.5` (диапазон `61+`)
- Java: `21`
- Тип: client-side (конфиг `bws_util-client.toml`)
- Внешний автокликер (опционально): [PLAYRUrk/AUT-CLK](https://github.com/PLAYRUrk/AUT-CLK) через локальный TCP bridge `127.0.0.1:25566`

### Интеграция с AUT-CLK

Этот мод поддерживает управление состоянием автокликера через bridge-протокол `AUT-CLK`:
- мод подключается к локальному bridge на `127.0.0.1:25566`;
- отправляет команды `ac_control` (включая выборочное управление `lmb_enabled` / `rmb_enabled`);
- получает `ac_status` для синхронизации состояния.

Чтобы интеграция работала:
1. Запустите `AUT-CLK`.
2. Убедитесь, что в GUI включен блок `Control bridge` (`Enable`, не `OFF`).
3. Запустите игру с этим модом — после подключения в `AUT-CLK` должен появиться статус `Bridge: Connected`.

## Быстрый старт

1. Собрать мод:
   - Windows: `.\gradlew.bat build`
2. Готовый JAR взять из `build/libs`.
3. Положить JAR в папку `mods` вашего Forge-клиента.
4. Запустить игру и настроить бинды/параметры прицела.

## Клавиши по умолчанию

- `Z` — открыть экран настроек прицела.
- `O` — включить/выключить прицел целиком.
- `M` — переключить режим `MANUAL` / `AUTO`.
- `=` / `-` — контекстное действие:
  - при активном прицеле в `MANUAL` — меняют дистанцию пристрелки (`+/-5`, `10..200`);
  - при включенном BedWars overlay и режиме `AUTO` (или когда лук не активен) — меняют масштаб радара.
- `B` — включить/выключить BedWars-оверлей.

## GUI: Scope Settings (экран настроек)

Открывается клавишей `Z`.

### Элементы экрана

- **Zero Distance Slider**
  - Диапазон: `10..200` блоков.
  - Меняет дистанцию, на которой стрела пересекает линию прицеливания.
- **Reticle Color Presets**
  - Набор быстрых цветов: Green, White, Red, Cyan, Yellow, Magenta.
  - Применяется сразу, без кнопки сохранения.
- **Checkbox: Show only while drawing**
  - Если включено, прицел виден только при натяжении лука.
- **Checkbox: Show range markers**
  - Включает/выключает стадиальные метки дистанции.
- **Checkbox: Show rangefinder**
  - Включает/выключает дальномер до блока/сущности в центре взгляда.
- **Checkbox: Show charge bar**
  - Включает/выключает вертикальный индикатор натяжения лука.
- **Close**
  - Закрывает экран настроек.

Экран не ставит игру на паузу.
Значения применяются сразу и сохраняются в клиентский конфиг.

## Основной оверлей прицела (Scope Overlay)

### Общие условия отображения

Прицел рисуется только если:
- игрок в мире,
- HUD не скрыт (`F1` выключен),
- прицел в состоянии `enabled`,
- в руках есть лук (или лук натягивается),
- и если включен флаг "Show only while drawing", то строго во время натяжения.

### Визуальные элементы прицела

- центральная метка,
- основной крест прицеливания с учетом пристрелки (`Zero Distance`),
- индикатор текущего режима (`[MAN]` / `[AUTO]`) справа,
- дополнительные элементы в зависимости от режима.
- темная обводка линий/текста для читаемости на любом фоне.

## Режимы прицела

## `MANUAL`

Режим ручной стрельбы с максимальной информацией:

- **Основной крест** — точка пристрелки под выбранную дистанцию.
- **Secondary charge mark** — дополнительная метка при неполном натяжении.
- **Stadia marks** — метки дистанции вокруг значения пристрелки.
- **Rangefinder** — расстояние до объекта в центре взгляда.
- **Zero label** — подпись текущей пристрелки (`⊕ Xm`).
- **Charge bar** (опционально) — прогресс натяжения.

Дополнительные детали отображения:
- дальномер показывает `◎ N.Nm`;
- цвет дальномера становится янтарным, если дистанция заметно отличается от `Zero`;
- стадиальные метки подсвечивают ближайшую к измеренной дистанции;
- крупные метки (дальние/ключевые) длиннее, промежуточные — короткие.

## `AUTO`

Режим автосопровождения:

- При натяжении лука происходит захват ближайшей `LivingEntity` в конусе около прицела.
- После захвата рассчитывается упреждение по горизонтальному движению цели.
- На экране рисуется ромб (`◇`) в точке, куда нужно наводиться.
- Рядом с ромбом рисуется подпись: `имя_цели + дистанция`.
- Учитывается:
  - расстояние до цели,
  - скорость цели между тиками,
  - баллистическая поправка под текущий заряд и пристрелку.
- Захват сбрасывается:
  - после выстрела/отмены натяжения,
  - при смене режима,
  - если цель перестала быть валидной.

## BedWars Overlay (B)

Тактическая панель в левом верхнем углу. Обновляется в реальном времени и частично по тикам.

### Что показывает

- Заголовок `BEDWARS` и количество hostile-игроков.
- **Danger line**:
  - тревога по ближайшему врагу в радиусе ~15м,
  - мигающий цвет предупреждения,
  - направление, дистанция и пометка `⚠BOW`, если враг натягивает лук.
- **ENEMIES**:
  - список ближайших врагов (ограниченный по количеству),
  - направление (`↑ ↓ ← → ↗ ↘ ↖ ↙`), имя, дистанция, броня и направление движения.
- **TEAM**:
  - союзники с теми же метриками.
- **TEAM POWER**:
  - сравнительная сила команд (кол-во игроков, агрегированный score, статусы),
  - отметки приоритетов (`HUNT`, `THRT`, `(you)`).
- **Статус кровати команды**:
  - `✔` жива, `✗` сломана, `?` не обнаружена,
  - код защиты: `UNK`, `---`, `WOL`, `TER`, `END`, `OBS`.
- **Тактический радар (правый верхний угол)**:
  - враги с направлением движения;
  - конструкции вокруг игрока (мосты/стены/высокие блоки разными цветами);
  - кольца дистанции;
  - вращается вместе со взглядом игрока;
  - масштаб меняется `=` / `-` по контекстной логике.

### Формат строки игрока (ENEMIES/TEAM)

Каждая строка содержит:
- стрелку направления до цели;
- укороченный ник (при натяжении лука у врага добавляется `!`);
- дистанцию в метрах (и разницу высоты `+N`/`-N`, если цель сильно выше/ниже);
- уровень брони в скобках: `N/D/I/C/G/L` (берётся по лучшему материалу из любой надетой части брони).

### Формат таблицы TEAM POWER

Для каждой команды показываются:
- цветовой маркер команды;
- сокращенное имя команды;
- количество игроков (`Np`);
- полоска силы команды;
- статус кровати и код застройки;
- правый тег приоритета:
  - `(you)` — ваша команда,
  - `→HUNT` — самая слабая вражеская команда,
  - `THRT` — самая сильная вражеская команда.

### Логика кроватей

- Каждые ~3 секунды выполняется скан области вокруг игрока.
- Находятся головы кроватей и оценивается защита вокруг.
- Для уже известных кроватей проверяется факт разрушения.
- Цвет команды сопоставляется с `DyeColor` для привязки кровати к команде.
- Для определения типа защиты учитывается локальная оболочка вокруг кровати; блоки ниже уровня кровати не учитываются (материал острова игнорируется).

Расшифровка кодов застройки кровати:
- `UNK` — кровать команды пока не найдена в радиусе сканирования;
- `---` — кровать найдена, но выраженной защиты нет;
- `WOL` — слабая защита;
- `TER` — средняя защита;
- `END` — сильная защита;
- `OBS` — очень сильная/тяжелая для взлома защита.

## Полная расшифровка обозначений GUI

### Режимы и статусы

- `[MAN]` — ручной режим прицела.
- `[AUTO]` — режим автосопровождения с упреждением.
- `⊕ Xm` — текущая пристрелка (zero distance) в блоках/метрах.
- `◎ N.Nm` — измеренная дальномером дистанция до блока/цели в центре взгляда.

### Направления

- `↑` — цель спереди.
- `↓` — цель сзади.
- `←` — цель слева.
- `→` — цель справа.
- `↖` / `↗` / `↙` / `↘` — диагональные направления.

### Обозначения в строке врага/тиммейта

- `!` после ника — игрок сейчас натягивает лук.
- `⚠BOW` — ближайший опасный враг натягивает лук (в danger-строке).
- `Nm` — горизонтальная дистанция до игрока.
- `+N` / `-N` рядом с дистанцией — цель выше / ниже вас по высоте.
- `[N] [D] [I] [C] [G] [L]` — лучший класс брони по надетым частям:
  - `N` — netherite,
  - `D` — diamond,
  - `I` — iron,
  - `C` — chainmail,
  - `G` — gold,
  - `L` — leather.

### TEAM POWER / приоритеты

- `Np` (например `3p`) — количество живых игроков команды.
- Полоска силы — относительная "мощь" команды (чем длиннее, тем сильнее).
- `(you)` — ваша команда.
- `→HUNT` — самая уязвимая вражеская команда (приоритет атаки).
- `THRT` — самая опасная вражеская команда (угроза).

### Статусы кровати

- `✔` — кровать жива.
- `✗` — кровать сломана.
- `?` — статус неизвестен (кровать еще не обнаружена в радиусе сканирования).

### Коды защиты кровати

- `UNK` — данных о кровати пока нет.
- `---` — кровать почти без защиты.
- `WOL` — слабая защита (шерсть/легкие блоки).
- `TER` — средняя защита (терракота/похожие блоки).
- `END` — сильная защита (часто endstone-уровень).
- `OBS` — очень прочная защита (обсидиан/аналог).

## Конфиг-файл

Файл: `config/bws_util-client.toml`

Основные параметры:
- `zero_distance` (`10..200`, default `50`)
- `reticle_color` (ARGB int)
- `show_only_while_drawing` (bool)
- `show_stadia_marks` (bool)
- `show_rangefinder` (bool)
- `show_charge_bar` (bool)

## Архитектура (кратко)

- `ScopeOverlay` — отрисовка прицела, режимы, дальномер, lead marker.
- `BedWarsOverlay` — тактическая BedWars-панель, скан кроватей, командная аналитика.
- `ClientGameEvents` — обработка клавиш и tick-логика (`ScopeOverlay.tick`, `BedWarsOverlay.tick`).
- `GuiRendererMixin` — точка отрисовки оверлеев поверх HUD.
- `ArrowPhysics` — баллистика стрелы (углы, падение, ticks полета).
- `ScopeConfigScreen` / `ScopeConfig` — GUI и хранение клиентских настроек.

## Проверка функциональности (чеклист)

1. Зайти в мир с луком.
2. Нажать `O`: прицел включается/выключается.
3. Нажать `M`: в правой части меняется индикатор `[MAN]`/`[AUTO]`.
4. Натянуть лук:
   - в `MANUAL` видны метки/дальномер по настройкам,
   - в `AUTO` появляется ромб упреждения на цели.
5. Проверить `=` / `-`:
   - в `MANUAL` с активным прицелом меняется `Zero`,
   - при активном BedWars overlay в `AUTO` или без активного лука меняется масштаб радара.
6. Нажать `Z`: открыть GUI настроек и проверить чекбоксы/слайдер/цвет.
7. Нажать `B`: включить/выключить BedWars-панель.
8. Убедиться, что `F1` скрывает весь HUD вместе с оверлеями.

## Известные ограничения

- AUTO-захват ограничен угловым конусом и может выбирать не ту цель в плотной толпе.
- Скорость цели в AUTO оценивается по клиентской дельте между тиками, при лагах точность снижается.
- Определение кроватей зависит от текущего радиуса сканирования вокруг игрока.
